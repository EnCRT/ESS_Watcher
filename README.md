# ESS Watcher (Energy Supply Status Watcher)

Проект умного мониторинга наличия сети 220В с оповещением в Telegram-бот при изменении состояния питания. 

---

## 📋 Описание проекта
Устройство постоянно отслеживает наличие напряжения в сети через входной разъем Type-C. При пропадании или появлении питания система отправляет уведомление в Telegram через Wi-Fi. Наличие встроенного аккумулятора 18650 обеспечивает автономную работу устройства и возможность отправки сообщения даже после отключения основного электричества.

**Ключевые (Pro) функции:**
- Мониторинг сети 220В в реальном времени.
- **Онлайн-конфигурация**: Настройка WiFi и Telegram через веб-интерфейс (режим точки доступа).
- Telegram-оповещения с поддержкой **Топиков супергрупп** (`message_thread_id`).
- Умная защита от перегрева чипа ESP32-C3.
- NTP-синхронизация времени для надежной проверки SSL/TLS сертификатов.
- Световая индикация состояния (RGB LED) и Звуковая индикация мелодиями (Passive Buzzer KY-006, формат RTTTL).
    - **Индикация состояния**: Двойной "пых" раз в 2 сек — зелёный (сеть 220В есть), красный (работа от АКБ), синий (режим настройки).
- Автономное питание от аккумулятора 18650.
- **Тихий режим**: настраиваемый интервал (например, ночью), когда зуммер молчит (отправляются только Telegram-сообщения).
- **Мягкий старт**: отложенная инициализация периферии для защиты boost-конвертера UPS.
- **Анти-коллизия тока**: мелодия и Telegram-сообщение не работают одновременно (очередь сообщений).
- **Детекция brownout**: устройство различает нормальный старт и перезагрузку после пропадания сети.

## ⚙️ Настройка (Configuration Mode)

Если устройство не может подключиться к сети или вам нужно изменить токены Telegram:
1. Зажмите кнопку **BOOT** (или внешнюю кнопку на **GPIO 9**) на 2 секунды.
2. Светодиод загорится **синим**, устройство создаст WiFi точку `ESS Watcher`.
3. Подключитесь к ней (пароль `12345678`).
4. Откройте в браузере `192.168.0.11` (или страница откроется автоматически).
5. Введите новые данные и нажмите **Сохранить**. Устройство перезагрузится.

## 📍 Схема подключения

| Компонент | Пин ESP32-C3 | Примечание |
|-----------|--------------|------------|
| **KY-006 Buzzer (Signal)** | GPIO 2 | Управление звуком через ШИМ (LEDC) |
| **WS2812B (DIN)** | GPIO 10 | Управление цветом |
| **Sense (Power Check)** | GPIO 0 | 5В с платы UPS → AMS1117-3.3 → 10кОм → GPIO 0 |
| **UPS Dummy Load** | — | Резистор 240 Ом на выходе 5В UPS (+/–) для предотвращения сна платы UPS |

> **Примечание:** В будущем планируется замена постоянного резистора на UPS на транзисторный ключ под управлением ESP32 — для кратковременной подачи нагрузки и сброса таймера сна без постоянного расхода энергии.

## 📂 Структура проекта
- `src/main.cpp` — основной код прошивки, включающий событийный Wi-Fi и HTTPClient логику.
- `include/BuzzerManager.h` — RTTTL-парсер и асинхронный проигрыватель мелодий для KY-006.
- `include/SoundStorage.h` — загрузчик звуков из файловой системы LittleFS.
- `include/ConfigManager.h` — хранение и доступ к настройкам в NVS.
- `include/WebPortal.h` — HTML/CSS/JS веб-интерфейса настройки.
- `include/secrets.example.h` — шаблон для конфигурации ваших паролей и токенов.
- `data/sounds/` — папка с RTTTL-мелодиями (системные и пользовательские звуки).
- `platformio.ini` — конфигурация сборки и библиотек.
- `extra_script.py` — авто-заливка LittleFS после прошивки.
- `assets/` — изображения компонентов и схемы.
- `DOCUMENTATION.md` — полная техническая документация.
- `TELEGRAM_SETUP.md` — инструкция по созданию Telegram-бота и получению chat_id.
- `README.md` — описание проекта.

## 🔒 Безопасность
Все конфиденциальные данные (Wi-Fi пароли, Telegram токены) вынесены в файл `include/secrets.h`, который **добавлен в `.gitignore`**. 
Перед компиляцией проекта скопируйте файл `secrets.example.h`, переименуйте его в `secrets.h` и впишите свои данные.
